# Task Manager App

A full-stack task management application built with **ASP.NET Core 9 Web API** (Clean Architecture) and **React 19** frontend. Created as a practical sandbox for testing backend architecture, data modeling, and design patterns in real code — evolved into a complete full-stack application with authentication, CRUD operations, Kanban board, analytics, and more.

## Performance Optimizations
- **Database indexes** — `Deadline`, `Status`, `Priority`, `UserId`, and composite `(Status, Priority)` indexes on `TaskItem` table for fast filtering and sorting
- **AsNoTracking** — read-only queries (`GetAllAsync`, `GetByIdAsync`, `GetPagedAsync`) use `AsNoTracking()` to skip EF Core change tracking overhead
- **Include(User)** — all task queries include the `User` navigation to prevent N+1 queries when mapping to `TaskDto` with `Username` field
- **Query filters** — global `HasQueryFilter(!IsDeleted)` ensures soft-deleted records are automatically excluded without manual `Where` clauses
- **Smart Priority Algorithm** — frontend sorting algorithm (`utils/deadline.ts`) that calculates a priority score for each task:
  - **Urgency score** (0–150): based on deadline proximity — overdue tasks get 100+, due today = 90, 3 days = 70, 1 week = 50, 2 weeks = 30, far future = 10
  - **Priority weight**: Critical = 4, High = 3, Medium = 2, Low = 1
  - **Total score** = urgency × priority — higher means more important
  - Used as default sort on Dashboard ("Smart Priority") and within Kanban columns
  - Algorithm inspired by [Eisenhower Matrix](https://en.wikipedia.org/wiki/Time_management#The_Eisenhower_Method) urgency-importance principle

## Code Sources & References

This project is built from scratch for educational purposes. The following resources were used as references for patterns, architecture, and specific implementations:

### Architecture & Patterns
- **Clean Architecture** — inspired by [Jason Taylor's Clean Architecture template](https://github.com/jasontaylordev/CleanArchitecture) (project layering, dependency rules)
- **DDD entity pattern** — private setters, factory methods (`Create`), domain methods (`UpdateDetails`, `SoftDelete`, `AssignTo`)
- **EF Core Fluent API configuration** — standard Microsoft documentation patterns for entity configuration
- **AutoMapper conditional mapping** — `ForAllMembers(condition)` pattern from AutoMapper documentation for partial updates

### Frontend Patterns
- **React 19 + TypeScript** — standard Vite template with strict TypeScript
- **TailwindCSS dark mode** — CSS variables approach from TailwindCSS documentation
- **Debounced search** — `useDebounce` hook pattern from [usehooks.com](https://usehooks.com/)
- **HTML5 drag & drop** — standard MDN Web API for Kanban board
- **react-markdown** — standard library for markdown rendering in task descriptions
- **Relative date formatting** — custom `utils/deadline.ts` utility inspired by Google Calendar's relative time display
- **Keyboard shortcuts overlay** — inspired by Obsidian's `?` shortcut panel pattern
- **Smart Priority Algorithm** — urgency × importance scoring inspired by [Eisenhower Matrix](https://en.wikipedia.org/wiki/Time_management#The_Eisenhower_Method), implemented in `utils/deadline.ts`

### Backend Patterns
- **JWT authentication with refresh tokens** — standard ASP.NET Core JWT pattern with BCrypt password hashing
- **Rate limiting** — ASP.NET Core 8+ built-in rate limiter with token bucket policy
- **Response caching** — standard `[ResponseCache]` attribute with `VaryByQueryKeys`
- **Soft delete with query filters** — EF Core `HasQueryFilter` for global soft-delete filtering
- **Optimistic concurrency** — EF Core `IsRowVersion()` with `RowVersion` byte array

### Database & Migrations
- **EF Core migrations** — the project uses `Database.MigrateAsync()` in `DbSeeder` on startup. Migrations are auto-generated by EF Core at runtime. To add new fields or tables:
  1. Add/modify the entity in `TaskManager.Domain/Entities/`
  2. Add/modify the Fluent API configuration in `TaskManager.Infrastructure/Persistence/EntityConfigurations/`
  3. Add/modify the DTO in `TaskManager.Application/DTOs/`
  4. Update the AutoMapper profile in `TaskManager.Application/Mapping/MappingProfile.cs`
  5. Add FluentValidation rules in `TaskManager.Application/Validators/` if needed
  6. The database schema will be auto-migrated on next startup via `MigrateAsync()`
- **PostgreSQL** — Npgsql EF Core provider, connection string in `appsettings.json` or environment variable `ConnectionStrings__DefaultConnection`

## Getting Started

### Prerequisites
- [.NET 9 SDK](https://dotnet.microsoft.com/download/dotnet/9.0)
- [PostgreSQL](https://www.postgresql.org/download/) (running on `localhost:5432`)
- [Node.js 20+](https://nodejs.org/) and npm

### Quick Start

**Windows:**
```batch
start.bat
```

**Linux:**
```bash
chmod +x start.sh
./start.sh
```

This will:
1. Restore NuGet packages
2. Build the solution
3. Start the backend API on `http://localhost:5000`
4. Install frontend dependencies and start Vite dev server on `http://localhost:3000`

### Manual Start

**Backend:**
```bash
cd TaskManagerProject
dotnet restore
dotnet build
cd TaskManager.API
dotnet run
```

**Frontend:**
```bash
cd TaskManagerProject/frontend
npm install
npm run dev
```

### Docker

```bash
docker-compose up --build
```

Services:
- PostgreSQL on `localhost:5432`
- Backend API on `localhost:5000`
- Frontend on `localhost:3000`

### Configuration
- Database connection string is in `appsettings.json` (use User Secrets for production)
- JWT secret key should be set via environment variable or User Secrets
- Frontend proxies `/api` requests to `http://localhost:5000` during development

### Authentication Flow

```mermaid
sequenceDiagram
    participant C as Client
    participant A as AuthController
    participant S as AuthService
    participant R as UserRepository
    participant T as TokenService

    C->>A: POST /api/v1/auth/login {email, password}
    A->>S: LoginAsync(request)
    S->>R: GetByEmailAsync(email)
    R-->>S: User (with BCrypt hash)
    S->>S: BCrypt.Verify(password, hash)
    alt Valid credentials
        S->>T: GenerateAccessToken(user)
        T-->>S: JWT token
        S->>T: GenerateRefreshToken()
        T-->>S: Refresh token
        S->>R: SaveRefreshTokenAsync()
        S-->>A: AuthResult {token, refreshToken, user}
        A-->>C: 200 OK {token, refreshToken, user}
    else Invalid credentials
        S-->>A: null
        A-->>C: 401 Unauthorized
    end

    Note over C,T: Token expires → use refresh

    C->>A: POST /api/v1/auth/refresh {refreshToken}
    A->>S: RefreshAsync(refreshToken)
    S->>R: ValidateRefreshTokenAsync()
    alt Valid refresh token
        S->>T: Generate new tokens
        S-->>A: New AuthResult
        A-->>C: 200 OK {new token, new refreshToken}
    else Invalid refresh token
        A-->>C: 401 Unauthorized
    end
```
